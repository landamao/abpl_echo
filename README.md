# 📢 复读器

**发送/复读指定内容，撤回引用消息，自动检测重复消息并复读或打断，并支持自动撤回自己发送的消息。**

---

## ✨ 功能一览

| 功能 | 说明 |
|------|------|
| 📝 `/发送 内容` | 机器人原样发送文字（别名：`/发`、`/文本`） |
| 🔁 `复读` | 引用一条消息后输入“复读”，机器人复读该消息 |
| 🗑️ `撤回` | 引用一条消息后输入“撤回”，机器人撤回被引用的消息 |
| 🦜 自动复读/打断 | 群内多人连续发送相同内容时，机器人自动复读或发送自定义打断语 |
| ⏱️ 自动撤回 | 本插件发送的所有消息（发送、复读、打断等）均可定时自动撤回 |

---

## ⚙️ WebUI 配置项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `复读机` | 开关 | 开启 | 启用自动检测重复消息功能 |
| `复读阈值` | 整数 | 2 | 连续多少条相同消息才触发复读/打断（建议2~10） |
| `复读模式` | 下拉选择 | 自定义内容 | `跟着复读` 或 `自定义内容` |
| `打断内容` | 文本 | 打断施法 | 当模式为“自定义内容”时发送的文字 |
| `发送` | 开关 | 开启 | `开启`=仅管理员可用 `/发送`；`关闭`=所有人可用 |
| `复读` | 开关 | 开启 | `开启`=仅管理员可用 `复读` 命令；`关闭`=所有人可用 |
| `撤回` | 开关 | 开启 | `开启`=仅管理员可用 `撤回` 命令；`关闭`=所有人可用 |
| `自动撤回` | 开关 | 开启 | 是否自动撤回本插件发送的消息（发送、复读、打断等） |
| `延迟时间` | 小数 | 20 | 发送消息后多少秒撤回（支持小数，如0.5） |

> 💡 修改配置**立即生效**，无需重启机器人。

---

## 📱 命令用法

### 1️⃣ 发送文本
```
/发送 你好呀
```
或 `/发 你好呀`、`/文本 你好呀`  
→ 机器人回复：`你好呀`

### 2️⃣ 复读引用消息
- 引用（回复）一条消息后输入：
```
复读
```
→ 机器人原样发送被引用的内容

### 3️⃣ 撤回引用消息
- 引用一条消息后输入：
```
撤回
```
→ 机器人尝试撤回被引用的消息（需有删除权限）

## 🦜 自动复读/打断（被动触发）

当群内连续出现 **N 条相同消息**（N = `复读阈值`）且**发送者不同**时，机器人的响应由 `复读模式` 决定：

| 复读模式 | 机器人行为 |
|----------|------------|
| `跟着复读` | 原样复读该消息 |
| `自定义内容` | 发送 `打断内容` 中设置的文字（默认“打断施法”） |

**触发规则**：
- 必须是不同人发送的相同纯文本消息，同一个人连发不计入
- 达到阈值后**只触发一次**，同一段消息不会重复触发
- 出现不同的消息后重置计数，开始新一轮检测
- 只处理纯文本消息，含图片、表情等富文本的消息会被忽略并重置计数

**配置示例**：
| 目标行为 | 复读阈值 | 复读模式 | 打断内容 |
|----------|----------|----------|----------|
| 两人发相同内容直接复读 | 2 | 跟着复读 | （忽略） |
| 三人发相同内容才复读 | 3 | 跟着复读 | （忽略） |
| 两人发相同内容就打断说“别复读了” | 2 | 自定义内容 | 别复读了 |
| 十人刷屏才警告“你们够了” | 10 | 自定义内容 | 你们够了 |

---

## ⏱️ 自动撤回

若启用 `自动撤回` 且 `延迟时间` > 0，则**本插件发送的所有消息**（包括 `/发送` 的内容、`复读` 的内容、打断消息等）将在指定秒数后自动删除。

> ⚠️ 注意：自动撤回仅对插件自身发出的消息有效，不会影响其他消息。

---

## 🔐 权限说明

- **管理员**始终可以使用所有命令，不受配置限制。
- **普通成员**的可用命令由下表决定：
  - `发送` = `关闭` → 普通成员可用 `/发送`
  - `复读` = `关闭` → 普通成员可用 `复读`
  - `撤回` = `关闭` → 普通成员可用 `撤回`

---

## ⚠️ 注意事项

1. **撤回引用消息**需要机器人有撤回对应消息的权限。
2. **自动复读机**不会复读机器人自己发的消息，也不会处理命令消息（如 `/发送 xxx`）。
3. 插件被禁用或重载时，所有未执行的自动撤回任务会被自动取消。
4. 自动复读机只处理纯文本消息，包含图片/表情等消息会重置计数。

## 🐛 常见问题

| 问题 | 解决 |
|------|------|
| `/发送` 没反应 | 检查配置 `发送` 是否限制了仅管理员，或消息内容为空 |
| 复读/撤回不生效 | 必须**引用（回复）**一条消息后再输入命令 |
| 自动撤回没执行 | 确认 `自动撤回` 为 `开启`，且 `延迟时间` > 0；检查机器人是否有删除消息权限 |
| 自动复读机不工作 | 确认 `复读机` 为 `开启`；检查 `复读模式` 和 `打断内容`；需要连续 N 条不同用户的相同纯文本消息（N = `复读阈值`） |
