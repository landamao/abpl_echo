# 复读器插件

发送/复读指定内容，撤回引用消息，自动检测重复消息并复读或打断。

## 功能

### 命令功能

- `/发送 内容` — 将指定内容原样发送（别名：`/发`、`/文本`）
- 引用消息后发送 `复读` — 复读被引用的消息
- 引用消息后发送 `撤回` — 撤回被引用的消息

### 自动复读/打断

群内多人连续发送相同消息时，根据配置自动处理：

- **跟着复读**：原样复读该消息
- **自定义内容**：发送自定义文字（默认：打断施法）

触发规则：
- 必须是不同人发送的相同消息才算，同一个人连发不会触发
- 达到「复读阈值」后触发一次，同一段消息只会触发一次
- 出现不同的消息后才会重置计数，开始新一轮检测
- 仅对纯文本消息生效，包含图片等富媒体的消息不参与计数

### 自动撤回

插件发送的消息支持自动撤回，可配置延迟时间。适用于复读、打断、发送文本等所有由本插件发出的消息。

## WebUI 配置项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 复读机 | 开关 | 开启 | 启用自动检测重复消息功能 |
| 复读阈值 | 数字 | 2 | 多少条相同消息才触发复读/打断（建议2-10） |
| 复读模式 | 下拉选择 | 自定义内容 | `跟着复读` 或 `自定义内容` |
| 打断内容 | 文本 | 打断施法 | 复读模式为自定义内容时发送的文字 |
| 发送 | 开关 | 开启 | 仅管理员可以使用 `/发送` 命令 |
| 复读 | 开关 | 开启 | 仅管理员可以使用引用复读 |
| 撤回 | 开关 | 开启 | 仅管理员可以使用引用撤回 |
| 自动撤回 | 开关 | 开启 | 是否自动撤回此插件发送的消息 |
| 延迟时间 | 数字 | 20 | 发送消息后多少秒后撤回（0-100） |

## 配置示例

- 阈值2 + 自定义内容「打断施法」：两个人发一样的就打断
- 阈值3 + 跟着复读：三个人发一样的才复读
- 阈值10 + 自定义内容「你们够了」：十个人发一样的才发打断消息
- 自动撤回开启 + 延迟30秒：插件发的消息30秒后自动消失

## 安装

将插件文件夹放入 `data/plugins/` 目录，在WebUI中启用插件即可。
