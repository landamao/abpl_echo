# 📢 复读器 插件使用文档

> 多功能复读工具：**发送文本**、**复读引用消息**、**撤回引用消息**、**自动复读机**，还能让机器人**自动撤回自己发的消息**。

---

## ✨ 功能概览

| 功能 | 说明 |
|------|------|
| 📝 `/发送 内容` | 让机器人原样发送一段文字（支持别名 `/发`、`/文本`） |
| 🔁 `复读` | 引用一条消息后输入“复读”，机器人复读该消息 |
| 🗑️ `撤回` | 引用一条消息后输入“撤回”，机器人撤回被引用的消息 |
| 🦜 自动复读机 | 群内连续两条相同消息（不同用户）时，机器人自动复读 |
| ⏱️ 自动撤回 | 机器人自己发送的消息（发送/复读）可在指定秒数后自动撤回 |

---

## ⚙️ 配置项（WebUI 可调）

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `发送` | bool | `true` | `true`=仅管理员可用 `/发送`；`false`=所有人可用 |
| `复读` | bool | `true` | `true`=仅管理员可用 `复读` 命令；`false`=所有人可用 |
| `撤回` | bool | `true` | `true`=仅管理员可用 `撤回` 命令；`false`=所有人可用 |
| `复读机` | bool | `true` | 是否启用自动复读机（检测连续相同消息） |
| `自动撤回` | bool | `true` | 是否自动撤回**本插件发送**的消息 |
| `延迟时间` | float | `20` | 自动撤回的延迟秒数（支持小数，如 `0.5`） |

> 💡 修改配置后**立即生效**，无需重启机器人。

---

## 📱 命令用法

### 1️⃣ 发送文本
```
/发送 你好呀
```
或
```
/发 你好呀
```
或
```
/文本 你好呀
```

- 机器人会原样回复：`你好呀`
- 若启用 `自动撤回`，消息将在 `延迟时间` 秒后被撤回。

### 2️⃣ 复读引用消息
- 引用（回复）一条消息，然后输入：
```
复读
```
- 机器人会**原样发送**被引用的消息内容。
- 若启用 `自动撤回`，机器人的复读消息会延迟撤回。

### 3️⃣ 撤回引用消息
- 引用（回复）一条消息，然后输入：
```
撤回
```
- 机器人会**撤回**被引用的那条消息（如果机器人有管理员权限可以撤回别人发的）。

### 4️⃣ 自动复读机（被动触发）
- 当群内**连续两条相同消息**（内容完全一致，且发送者不同）时，机器人会自动复读一次。
- 示例：
  > 用户A：哈哈哈<br>
  > 用户B：哈哈哈<br>
  > 机器人：哈哈哈

---

## 🔧 权限说明

- **管理员**始终可以使用所有命令，不受配置限制。
- **普通成员**的可用命令由配置决定：
  - `发送` = `false` → 普通成员可用 `/发送`
  - `复读` = `false` → 普通成员可用 `复读`
  - `撤回` = `false` → 普通成员可用 `撤回`

---

## ⚠️ 注意事项

1. **自动撤回仅对本插件发送的消息生效**（即 `/发送` 和 `复读` 命令产生的消息），不会撤回其他消息。
2. 撤回功能需要机器人拥有 **删除消息** 的权限（群管理员或授权）。
3. 自动复读机**不会复读机器人自己发的消息**，也不会复读命令消息（如 `/发送 xxx`）。
4. 插件被禁用或重载时，所有未执行的自动撤回任务会被自动取消。

## 🐛 常见问题

| 问题 | 解决 |
|------|------|
| `/发送` 没反应 | 检查配置 `发送` 是否限制了仅管理员，或消息内容为空 |
| 复读/撤回不生效 | 必须**引用（回复）** 一条消息后再输入命令 |
| 自动撤回没执行 | 确认 `自动撤回` 为 `true`，且 `延迟时间` > 0；检查机器人是否有删除消息权限 |
| 自动复读机不工作 | 确认 `复读机` 为 `true`；需要**连续两条相同消息**且发送者不同 |
